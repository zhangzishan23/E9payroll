1. 项目概述

1.1 项目名称

E9 智能化薪资核算系统

1.2 项目简介

面向 E9 公司（含北京易玖、广州分公司、邯郸分公司、上海瑞方 4 个法律主体）的薪资核算管理系统。系统以人事信息登记为基础，联动考勤、绩效、社保公积金等数据，自动完成每月薪资核算，替代当前手工 Excel 操作模式，提升核算效率与准确性。

1.3 核心目标

  • 数据一体化：人事信息一次录入，多表联动，消除重复劳动
  • 核算自动化：自动匹配数据源，按规则计算薪资，减少人工出错
  • 流程规范化：内置审批流程，确保薪资核算各环节可追溯、可管控
  • 权限精细化：admin 绝对特权，其余角色严格隔离，保障数据安全

2. 背景与现状

2.1 公司规模

  • 在职员工约 61 人，分布在 4 个法律主体；员工类型：实习生、试用期员工、正式员工、外包员工。

2.2 当前痛点

痛点描述
手工操作人事专员需整合多源 Excel 数据，手动复制粘贴、匹配公式
效率低下人事月报制作耗时 2-3 天，薪资核算耗时 3-5 天
易出错特殊情况（贺金、未报税奖金等）需人工记忆处理，多重审核仍难保证准确性
数据分散花名册、考勤表、绩效表、社保表各自独立，信息重复录入
缺乏权限控制无法按角色控制数据查看范围，其他部门获取人事数据不便

2.3 当前薪资核算流程（手工模式）

每月由人事专员根据人事月报进行基本工资核算，人事主管提供绩效评分数据，核算总应发工资后，提交财务申报个税，最终实发工资支付等环节。数据来源包括花名册、考勤表、绩效评分表、社保公积金表、遗留金额调整、临时性差旅补贴等。

2.4 现有 Excel 模板

模板名称用途备注
花名册全量员工基本信息表从钉钉导出
考勤表每月考勤统计含计薪天数、出勤率
绩效评分表月度绩效评价初评+复评得出绩效系数
薪资核算表月度薪资计算核心工资表
社保公积金表月度社保公积金含个人/公司缴纳金额

2.5 现有薪资计算公式

公式名称计算公式
月薪标准基本工资 + 绩效奖金标准 + 餐补 + 交通补 + 通讯补 + 电脑补 + 住房补
实际绩效奖金绩效奖金标准 × 绩效系数
总应发工资(基本工资 + 补贴合计 + 实际绩效奖金 + 提成/项目奖金/补发 + 税前调整金额) × 出勤率
本月实际报税金额本月总应发工资 + 上月未报税金额 + 临时性差旅补贴未报税费用
实发工资总应发工资 - 社保公积金个人缴纳合计 - 本月应扣个税 + 税后调整金额

4. 用户角色与权限

4.1 权限模型设计

三层权限模型：用户 → 角色 → 功能权限 + 数据范围权限，支持字段级权限。

4.2 预设角色

角色职责功能权限数据范围字段权限
admin系统管理员全部功能全部数据全部字段
人事专员主要操作者员工档案、考勤、薪资核算等全公司全部字段
人事主管绩效/审核绩效录入、薪资审核等全公司全部字段
人事经理终审薪资终审、报表查看全公司全部字段
会计差旅/个税差旅录入、个税查看/导出全公司仅金额相关字段
普通员工查看个人薪资工资条查看仅本人数据仅个人薪资字段

权限规则: admin绝对特权，其余最小必要权限，支持多角色并集，临时授权等。

4.3 权限配置功能

  • 角色管理、功能权限分配、数据范围设置、字段级权限、权限预览和继承。

5. 功能模块

5.1 人事信息登记模块

员工档案管理（增删改查/批量导入）、人事异动管理（入职转正调动调薪离职）、人事月报生成、多版本表格导出（花名册、考勤表、绩效评分表等）。

5.2 考勤管理模块

考勤数据导入、异常考勤识别（在家打卡标记）、考勤统计、计薪天数计算及出勤率自动生成。

5.3 薪资核算模块（核心）

薪资结构定义、数据源准备（花名册、人事月报、社保公积金、遗留调整、差旅补贴、绩效评价等），核算流程10个步骤，特殊处理规则完整参照文档。

类别项目说明
固定部分基本工资/绩效奖金标准/餐补/交通补/通讯补/电脑补/住房补固定月薪组成部分
变动部分实际绩效奖金/提成/项目奖金补发/税前调整/税后调整依系数或特殊调整

核算公式：月薪标准 = 基本工资 + ...；实际绩效 = 绩效标准 × 绩效系数；总应发 = (基本工资+补贴+实际绩效+提成+税前调整)×出勤率；实发 = 总应发-社保个税-个扣+税后调整。

5.4 审批流程模块

人事专员提交 → 人事主管审核 → 人事经理终审，支持驳回及审核留痕。

5.5 报表与导出模块

支持花名册、人事月报、考勤表、绩效评分表、薪资核算表、社保公积金表、工资条导出，且按权限控制金额字段。

5.6 系统管理模块

用户管理、角色权限管理（功能/数据/字段）、操作日志（登录/操作/数据变更，保留策略），数据字典管理（合同公司、部门、职务、用工状态、假期类型等），模块开关与生命周期管理。

5.7 模块架构与通信规范

模块间仅通过共享数据表通信，禁止直接代码引用。支持模块开关（关闭后前端隐藏，数据保留）。

6. 数据字典

系统包含员工表(employees)、考勤记录表、员工薪资表、薪资核算表、绩效评分表、社保公积金表、遗留金额调整表、临时差旅补贴表、劳动补偿金表、审核记录表、系统用户表、角色表、权限表、操作日志及数据字典基础表等。字段定义严格参照原文档。

7. 业务规则

  • 核算周期：每月10日启动，14日前完成个税申报，15日前支付工资。
  • 转正人员分段计算，补偿金单独报税，上月未报税金额自动累加，调休折算待定等。
  • 人事信息变更联动同步至考勤和薪资表。

8. 非功能需求

  • 性能：页面≤3秒，61人核算≤5秒，Excel批量导入≤200条。
  • 安全：bcrypt加密，JWT认证，操作日志全记录。
  • 技术栈：Vue3 + Tailwind CSS，Python FastAPI，PostgreSQL，阿里云百炼(Qwen)大模型，仅限GET/POST方法。
  • AI体验：流式输出；权限admin绝对特权；金额底层单位元，前端可展示万单位。
  • 部署：支持docker-compose一键部署，Windows优先，提供DEPLOY.md。
  • 错误处理全部中文，错误码规范1000~6999区间清晰定义。

9. 验收标准

  • 人事信息登记模块：可增删改查、批量导入、入转调离自动更新等
  • 考勤管理模块：导入钉钉报表，识别在家打卡，自动计算出勤率
  • 薪资核算模块：自动汇总7类数据源、人员变动，按公式计算总应发/实发，支持个税明细导入
  • 审批流程模块：三级审核，审核记录留痕
  • 系统管理模块：用户管理、角色权限自定义、日志、数据字典可配置
  • 权限控制：admin绝对特权，角色隔离及金额权限管控
  • 模块架构：独立开关，模块间通过共享表通信
  • 技术验收：前端Vue3+Tailwind，后端FastAPI，仅GET/POST，Swagger文档，docker部署，流式AI
