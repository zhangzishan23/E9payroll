# E9 智能化薪资核算系统 需求文档

## 1. 项目概述

### 1.1 项目名称

E9 智能化薪资核算系统

### 1.2 项目简介

面向 E9 公司（含北京易玖、广州分公司、邯郸分公司、上海瑞方 4 个法律主体）的薪资核算管理系统。系统以人事信息登记为基础，联动考勤、绩效、社保公积金等数据，自动完成每月薪资核算，替代当前手工 Excel 操作模式，提升核算效率与准确性。

### 1.3 核心目标

- **数据一体化**：人事信息一次录入，多表联动，消除重复劳动
- **核算自动化**：自动匹配数据源，按规则计算薪资，减少人工出错
- **流程规范化**：内置审批流程，确保薪资核算各环节可追溯、可管控
- **权限精细化**：admin 绝对特权，其余角色严格隔离，保障数据安全

## 2. 背景与现状

### 2.1 公司规模

- 在职员工约 **61 人**
- 分布在 **4 个法律主体**：北京易玖、广州分公司、邯郸分公司、上海瑞方
- 员工类型：实习生、试用期员工、正式员工、外包员工

### 2.2 当前痛点

| 痛点     | 描述                                      |
| ------ | --------------------------------------- |
| 手工操作   | 人事专员需整合多源 Excel 数据，手动复制粘贴、匹配公式          |
| 效率低下   | 人事月报制作耗时 2-3 天，薪资核算耗时 3-5 天             |
| 易出错    | 特殊情况（贺金、未报税奖金、调休折算等）需人工记忆处理，多重审核仍难保证准确性 |
| 数据分散   | 花名册、考勤表、绩效表、社保表各自独立，信息重复录入              |
| 缺乏权限控制 | 无法按角色控制数据查看范围，其他部门获取人事数据不便              |

### 2.3 当前薪资核算流程（手工模式）

每月由人事专员根据人事月报进行基本工资核算，人事主管负责提供绩效评分数据，核算出总应发工资后，提交给财务去税务局申报个税。导出个税表后，核算出实发工资，通过易快报申请费用并跟进审批流程，最终由出纳支付。支付后制作工资条并通过钉钉"智能工资条"发放。

当前用于制作工资表的数据来源包括：

- 花名册（从钉钉导出）
- 考勤表（从钉钉导出，手工核查）
- 绩效评分表（人事主管提供）
- 社保公积金表（人事专员维护）
- 遗留金额调整（历史错漏记录）
- 临时性差旅补贴报销（会计提供）
- 劳动补偿金报税（单独申请）

若绩效分开发放，则需重复以上工作，耗时翻倍。

### 2.4 现有 Excel 模板

| 模板名称   | 用途        | 备注          |
| ------ | --------- | ----------- |
| 花名册    | 全量员工基本信息表 | 从钉钉导出       |
| 考勤表    | 每月考勤统计    | 含计薪天数、出勤率   |
| 绩效评分表  | 月度绩效评价    | 初评+复评得出绩效系数 |
| 薪资核算表  | 月度薪资计算    | 核心工资表       |
| 社保公积金表 | 月度社保公积金   | 含个人/公司缴纳金额  |

### 2.5 现有薪资计算公式

本系统须完全复现当前 Excel 中的计算逻辑：

| 公式名称     | 计算公式                                               |
| -------- | -------------------------------------------------- |
| 月薪标准     | 基本工资 + 绩效奖金标准 + 餐补 + 交通补 + 通讯补 + 电脑补 + 住房补         |
| 实际绩效奖金   | 绩效奖金标准 × 绩效系数                                      |
| 总应发工资    | (基本工资 + 补贴合计 + 实际绩效奖金 + 提成/项目奖金/补发 + 税前调整金额) × 出勤率 |
| 本月实际报税金额 | 本月总应发工资 + 上月未报税金额 + 临时性差旅补贴未报税费用                   |
| 实发工资     | 总应发工资 - 社保公积金个人缴纳合计 - 本月应扣个税 + 税后调整金额              |

## 4. 用户角色与权限

### 4.1 权限模型设计

本系统采用**三层权限模型**：用户 → 角色 → 功能权限 + 数据范围权限

```
用户（User）
  ├── 关联角色（可多角色）
  │     ├── 角色A → 功能权限集合 + 数据范围
  │     └── 角色B → 功能权限集合 + 数据范围
  └── 直接权限（可选，用于特殊授权）
```

**权限层级说明**：

- **功能权限**：控制用户能访问哪些功能（如"员工档案-查看"、"薪资核算-编辑"）
- **数据范围权限**：控制用户能查看哪些数据（如"仅本部门"、"全公司"、"指定分公司"）
- **字段级权限**：控制用户能查看哪些字段（如"金额字段可见/隐藏"）

### 4.2 预设角色（可自定义修改）

系统预置以下角色作为初始模板，admin 可在系统中自由修改、新增、删除角色：

| 角色        | 职责        | 功能权限                                    | 数据范围  | 字段权限    |
| --------- | --------- | --------------------------------------- | ----- | ------- |
| **admin** | 系统管理员     | 全部功能                                    | 全部数据  | 全部字段    |
| **人事专员**  | 系统主要操作者   | 员工档案(增删改查)、考勤管理(导入/查看)、薪资核算(操作/导出)、报表导出 | 全公司   | 全部字段    |
| **人事主管**  | 绩效评分、薪资审核 | 绩效数据(录入/查看)、薪资审核(查看/通过/驳回)、部分报表查看       | 全公司   | 全部字段    |
| **人事经理**  | 最终审核      | 薪资终审(查看/通过/驳回)、报表查看                     | 全公司   | 全部字段    |
| **会计**    | 差旅补贴、个税申报 | 差旅补贴(录入/查看)、个税数据(查看/导出)                 | 全公司   | 仅金额相关字段 |
| **普通员工**  | 查看个人薪资    | 工资条(查看)                                 | 仅本人数据 | 仅个人薪资字段 |

> **权限规则**：
>
> 1. admin 拥有绝对特权（含系统配置与数据回滚），不可被其他角色限制
> 2. 其余角色按"最小必要权限"原则配置，admin 可逐个功能分配/禁用
> 3. 新增功能必须同步纳入权限配置体系，并测试权限隔离效果
> 4. 金额数据默认隐藏，仅限有权限的角色查看
> 5. 支持用户关联多个角色，权限取并集
> 6. 支持临时授权（如某人休假期间临时授予审核权限）

### 4.3 权限配置功能

admin 可在系统中进行以下权限配置：

- **角色管理**：新增/编辑/删除角色，设置角色描述
- **功能权限分配**：按功能模块逐个勾选（查看/新增/编辑/删除/导出）
- **数据范围设置**：全部数据 / 本部门及下级部门 / 仅本人 / 自定义部门列表
- **字段级权限**：对敏感字段（如身份证号、银行卡号、金额）设置可见/隐藏/脱敏
- **权限预览**：配置后可预览该角色实际拥有的权限清单
- **权限继承**：支持角色继承（如"高级人事专员"继承"人事专员"全部权限并追加）

## 5. 功能模块

### 5.1 人事信息登记模块

#### 5.1.1 员工档案管理

- 支持录入/导入员工基本信息：
  - 员工编号（系统自动生成，唯一）
  - 姓名、性别、身份证号、联系方式
  - 合同公司（北京易玖 / 广州分公司 / 邯郸分公司 / 上海瑞方）
  - 部门、职务
  - 用工状态（实习 / 试用期 / 正式 / 外包 / 离职）
  - 入职时间、转正时间、离职时间
  - 银行卡号、银行卡开户支行
  - 费用负责人
  - 家庭住址（用于考勤异常识别）
- 支持批量导入（Excel 导入）
- 支持单个新增、编辑、查看

#### 5.1.2 人事异动管理

- **入职**：录入新员工完整档案信息
- **转正**：更新员工状态为"正式"，可同步调整薪资
- **调动**：支持部门调动、岗位调动、合同主体调动
- **调薪**：记录调薪历史（基本工资、绩效奖金标准、各项补贴）
- **离职**：更新员工状态为"离职"，记录离职时间
- 所有异动记录自动归档到员工薪资档案

#### 5.1.3 人事月报生成

- 根据当月异动数据，自动生成人事月报，包含以下子表：
  - 入职人员表
  - 转正人员表
  - 异动人员表（调动/调薪）
  - 离职人员表
  - 公司间人员变动表

#### 5.1.4 多版本表格导出

- 花名册（全量员工信息）
- 考勤表（用于考勤统计）
- 绩效评分表（用于绩效评价）
- 薪资核算表（用于薪资计算）
- 社保公积金表（用于社保公积金核对）

### 5.2 考勤管理模块

#### 5.2.1 考勤数据导入

- 支持导入钉钉考勤导出报表
- 支持导入外勤打卡记录

#### 5.2.2 异常考勤识别

- 自动匹配员工家庭住址，识别在家打卡的异常情况
- 标记需与部门领导核实的记录

#### 5.2.3 考勤统计

- 跟进缺卡情况，标记需补卡员工
- 核实迟到、早退情况
- 根据员工手册规定，超标部分自动按事假处理
  - 【待确认】迟到/早退多少次折算为事假？折算比例是多少？
- 假勤情况核实后自动更新

#### 5.2.4 计薪天数计算

- 自动计算总计薪天数（当月应出勤天数）
- 自动计算实际计薪天数（扣除请假等）
- 自动计算出勤率 = 实际计薪天数 / 总计薪天数

### 5.3 薪资核算模块（核心）

#### 5.3.1 薪资结构定义

薪资由**固定部分**和**变动部分**组成：

| 类别   | 项目         | 说明               |
| ---- | ---------- | ---------------- |
| 固定部分 | 基本工资       | 员工月基本工资          |
| 固定部分 | 绩效奖金标准     | 绩效奖金基数           |
| 固定部分 | 餐补         | 每月固定餐费补贴         |
| 固定部分 | 交通补        | 每月固定交通补贴         |
| 固定部分 | 通讯补        | 每月固定通讯补贴         |
| 固定部分 | 电脑补        | 每月固定电脑补贴         |
| 固定部分 | 住房补        | 每月固定住房补贴         |
| 变动部分 | 实际绩效奖金     | = 绩效奖金标准 × 绩效系数  |
| 变动部分 | 提成/项目奖金/补发 | 按实际情况录入          |
| 变动部分 | 税前调整金额     | 特殊情况调整（含贺金、慰问金等） |
| 变动部分 | 税后调整金额     | 特殊情况调整（如调休折算扣款等） |

> 【待确认】各项补贴标准具体金额是多少？按岗位区分还是全公司统一？

#### 5.3.2 工资核算初始化

**功能入口**：薪资核算模块首页

**月份选择界面**：
- 默认自动选中上月（如当前为 5 月，则默认选中 4 月）
- 可选月份范围：最早为公司首个薪资核算月份，最晚为当前月份
- 已生成工资条的月份置为不可选状态（灰色显示），仅提供"查看"按钮
- 未核算月份显示"待核算"标识，已核算未发放月份显示"核算完成"标识

**核算按钮**：
- 位置：月份选择器右侧
- 点击后触发核算流程，进入"数据源管理与补充"界面
- 若该月份已初始化，则显示"重新核算"按钮（需二次确认）

#### 5.3.3 数据源管理与补充机制

**自动拉取数据**：
系统自动汇总以下数据源，并在界面左侧显示数据源清单：

| 序号 | 数据源          | 数据来源       | 说明                           |
| -- | ------------ | ---------- | ---------------------------- |
| 1  | 花名册          | 系统 employees 表 | 当前全量在职员工信息（状态≠离职）       |
| 2  | 员工薪资档案      | 系统 employee_salaries 表 | 最新生效的薪资标准（基本工资、绩效、补贴） |
| 3  | 考勤数据         | 系统 attendance_records 表 | 当月考勤统计（计薪天数、出勤率）        |
| 4  | 绩效系数         | 系统 performance_scores 表 | 当月绩效评价结果（绩效系数）           |
| 5  | 社保公积金       | 系统 social_insurance 表 | 当月社保公积金个人缴纳金额            |
| 6  | 遗留金额调整      | 系统 legacy_adjustments 表 | 历史错漏需本月补救的金额（税前/税后）     |
| 7  | 差旅补贴报销      | 系统 travel_reimbursements 表 | 当月员工差旅报销费用（需报税部分）       |
| 8  | 劳动补偿金       | 系统 labor_compensations 表 | 单独报税的经济补偿金（工资表单独列示）    |

**数据完整性校验**：
- 系统自动校验每位员工的数据完整性，校验规则：
  - ✅ 必需数据：员工档案、薪资档案、考勤数据、社保公积金
  - ⚠️ 可选数据：绩效系数（默认为 1.0）、差旅补贴、补偿金
- 界面右侧显示"数据完整性面板"，分为三个区域：
  - **完整数据**（绿色）：所有必需数据齐全的员工数量
  - **缺失数据**（红色）：缺少必需数据的员工清单，标注缺失项
  - **待补充数据**（黄色）：缺少可选数据的员工清单

**缺失数据补充界面**：
- 点击"缺失数据"中的员工姓名，弹出补充表单
- 支持三种补充方式：
  1. **手动录入**：直接在表单中填写缺失字段
  2. **Excel 导入**：下载模板 → 填写后上传 → 系统自动匹配
  3. **批量导入**：支持一次性导入多人数据（如绩效系数表）
- 补充完成后，系统实时刷新完整性状态
- 所有补充操作记录到操作日志

**确认按钮**：
- 当所有必需数据完整后，"确认并核算"按钮激活（蓝色）
- 点击后进入"一键核算"环节

#### 5.3.4 薪资计算流程（一键核算）

**一键核算触发**：
- 用户确认数据完整性后，点击"确认并核算"按钮
- 系统弹出二次确认框："确认开始核算 [YYYYMM] 月份薪资？核算过程约需 X 秒。"

**核算过程**：
系统按以下顺序自动计算（后台事务处理）：

1. **人员范围确定**：
   - 提取当月所有在职员工（含新入职、转正、调动、离职）
   - 按员工状态标记（试用/正式/实习/外包/离职）

2. **基础数据加载**：
   - 加载员工薪资档案（基本工资、绩效奖金标准、各项补贴）
   - 加载考勤数据（出勤率）
   - 加载绩效系数（默认 1.0）
   - 加载社保公积金个人缴纳金额

3. **应发工资计算**（按员工逐行计算）：
   - 月薪标准 = 基本工资 + 绩效奖金标准 + 各项补贴合计
   - 实际绩效奖金 = 绩效奖金标准 × 绩效系数
   - 补贴合计 = 餐补 + 交通补 + 通讯补 + 电脑补 + 住房补
   - **总应发工资** = (基本工资 + 补贴合计 + 实际绩效奖金 + 提成/项目奖金/补发 + 税前调整金额) × 出勤率
   - 注：提成/项目奖金/补发、税前调整金额初始为 0，可在核算结果界面手动调整

4. **特殊场景处理**：
   - **转正人员**：按转正日期分段计算（系统自动识别转正日期，按天数权重计算）
   - **新入职人员**：按实际计薪天数比例计算
   - **离职人员**：按实际计薪天数比例计算，自动检查未结调休

5. **数据写入**：
   - 将计算结果写入 salary_calculations 表（状态：草稿）
   - 生成核算日志（见下文）

**核算日志记录**：
系统自动生成核算日志，记录到 `calculation_logs` 表：
- 核算批次号（唯一标识）
- 核算周期（YYYYMM）
- 开始时间、结束时间、耗时
- 参与核算员工数
- 成功/失败记录数
- 详细日志（JSON 格式，包含每位员工的计算过程快照）
- 操作人 ID

**核算结果展示**：
- 核算完成后，自动跳转到"核算结果预览"界面
- 显示汇总统计：
  - 员工总数、核算成功数、异常数
  - 总应发工资合计（万元为单位）
  - 人均应发工资
- 表格展示每位员工的明细数据（可编辑）
- 顶部显示"本月总应发工资"大字提示

**此阶段输出**：
- ✅ 生成每位员工的**总应发工资**
- ✅ 生成核算日志（可追溯）
- ❌ 尚未计算实发工资（需等待个税数据上传）

#### 5.3.5 报税与实发工资核算

**综合所得数据上传**（前置条件）：
- 位置：核算结果预览界面顶部，显示"个税申报"按钮
- 操作流程：
  1. 系统导出"报税数据表"（Excel 格式），包含：
     - 员工姓名、身份证号、本月实际报税金额
     - 本月实际报税金额 = 总应发工资 + 上月未报税金额 + 临时性差旅补贴未报税费用
  2. 会计在税务局个税系统完成申报
  3. 从个税系统下载"综合所得申报明细表"（Excel）
  4. 在系统上传该 Excel 文件
- 系统自动解析上传的 Excel，匹配员工并更新以下字段：
  - 专项扣除（子女教育、房贷利息等）
  - 本月应扣个税（系统根据个税表自动计算或导入）

**实发工资自动核算**：
- 上传完成后，点击"计算实发工资"按钮
- 系统按以下公式逐行计算：
  - **实发工资** = 总应发工资 - 社保公积金个人缴纳合计 - 本月应扣个税 + 税后调整金额
- 计算结果实时更新到 salary_calculations 表
- 生成第二条核算日志（标记为"实发工资核算"）

**正式工资条生成**：
- 实发工资计算完成后，"生成工资条"按钮激活
- 点击后系统为每位员工生成工资条记录（salary_slips 表）
- 工资条状态：待审核
- 支持预览、导出 PDF、对接钉钉智能工资条

### 5.4 审批流程模块

#### 5.4.1 薪资审核流程

```
人事专员核算 → 批量提交审核（生成审批流水）
              ↓
        人事主管审核 → 驳回（附意见）→ 人事专员修改 → 重新提交
              ↓ 通过
        人事经理终审 → 驳回（附意见）→ 人事专员修改 → 重新提交
              ↓ 通过
          出纳支付
```

#### 5.4.2 审批提交

**批量提交审核**：
- 薪资核算页面提供"批量提交审核"按钮，勾选多条记录后一键提交
- 提交时生成审批流水号（格式：AP-YYYYMMDD-XXXX），记录提交人、提交时间
- 提交后该周期所有核算记录状态变更为"审核中"，不可再修改核算数据
- 同一周期只能有一个进行中的审批流程

**审批流水信息**：
- 流水号：AP-YYYYMMDD-XXXX（如 AP-20260515-0001）
- 核算周期：YYYYMM
- 提交人：提交审核的操作人姓名
- 提交时间：提交审核的具体时间
- 审批进度：待主管审核 / 主管已通过，待经理审核 / 经理已通过 / 已驳回
- 审核状态：审核中 / 已通过 / 已驳回
- 参与人数：本次提交审核的员工数量
- 总应发金额：本次提交审核的总应发工资

#### 5.4.3 审核功能

**审批人视图**：
- 审批人可查看完整工资表，包含所有核算字段
- 支持按员工编号、姓名筛选
- 支持逐行核对、标记疑问
- 显示审批流程进度条（提交 → 主管审核 → 经理审核 → 完成）

**审核操作**：
- 审核通过/驳回，附审核意见
- 驳回时必须填写驳回意见
- 审核记录留痕（谁、何时、什么操作、意见）
- 审核通过后，下一环节审批人可继续审核

#### 5.4.4 审批记录

| 字段名            | 中文名   | 类型           | 必填 | 说明             |

| id             | 主键    | INTEGER      | 是  | 自增             |

| submitter_id   | 提交人 ID | INTEGER      | 是  | 外键 → sys_users.id |

### 5.7 模块架构与通信规范

> 本节定义系统模块间的通信方式和生命周期管理规则，确保模块独立、可插拔、可关闭。

#### 5.7.1 模块分类

| 类别   | 模块                         | 说明               |
| ---- | -------------------------- | ---------------- |
| 业务模块 | 人事信息登记、考勤管理、薪资核算、审批流程、报表导出 | 直接面向业务场景         |
| 系统模块 | 用户管理、权限管理、操作日志、数据字典        | 系统基础设施，为业务模块提供支撑 |

#### 5.7.2 模块间通信规则

**核心原则：模块间仅通过共享数据表通信，严禁直接引用其他模块的代码（Service/State/Controller）**

- **共享表定义**：所有模块共用的数据表，如员工表(employees)、薪资核算表(salary\_calculations)
- **私有表定义**：仅单个模块使用的数据表，如操作日志表(operation\_logs)
- **通信方式**：
  - 模块 A 需要模块 B 的数据 → 直接查询共享表，不调用模块 B 的接口
  - 模块 A 需要触发模块 B 的动作 → 通过事件总线(Event Bus)发布事件，模块 B 监听并处理
  - 禁止模块 A 直接 import 模块 B 的类或方法

**共享表清单**：

| 共享表                               | 主要读取方       | 主要写入方    |
| --------------------------------- | ----------- | -------- |
| employees（员工表）                    | 所有业务模块      | 人事信息登记模块 |
| employee\_salaries（员工薪资表）         | 薪资核算模块、报表模块 | 人事信息登记模块 |
| salary\_calculations（薪资核算表）       | 审批流程模块、报表模块 | 薪资核算模块   |
| attendance\_records（考勤记录表）        | 薪资核算模块      | 考勤管理模块   |
| performance\_records（绩效记录表）       | 薪资核算模块      | 人事信息登记模块 |
| social\_security\_records（社保公积金表） | 薪资核算模块      | 人事信息登记模块 |

#### 5.7.3 模块生命周期管理

**模块开关机制**：

- 每个模块在系统配置表中有一个开关字段（enabled: true/false）
- admin 可在系统设置中逐个开启/关闭模块
- 关闭模块后：
  - 前端隐藏该模块的所有入口（菜单、按钮、链接）
  - 停止该模块的后台任务（定时任务、消息监听等）
  - **保留该模块的所有数据**，不删除任何记录
  - 其他模块对该模块数据的读取返回"模块已关闭"提示
- 重新开启模块后：
  - 恢复前端入口
  - 启动后台任务
  - 数据完整可用

**模块依赖关系**：

| 模块     | 依赖模块     | 依赖类型              |
| ------ | -------- | ----------------- |
| 薪资核算模块 | 人事信息登记模块 | 强依赖（人事数据是薪资核算基础）  |
| 薪资核算模块 | 考勤管理模块   | 强依赖（考勤数据影响出勤率）    |
| 薪资核算模块 | 审批流程模块   | 弱依赖（可先核算后审批）      |
| 审批流程模块 | 薪资核算模块   | 强依赖（审批对象是薪资表）     |
| 报表导出模块 | 所有业务模块   | 弱依赖（模块关闭时跳过该模块报表） |

## 7. 业务规则

### 7.1 核算周期

- 每月 **10 日**启动当月薪资核算
- 每月 **14 日**前完成个税申报（遇周末提前）
- 每月 **15 日**前完成工资支付（遇周末提前）

### 7.2 特殊处理规则

1. **转正人员薪资**：按转正日期分段计算（转正前按试用薪资，转正后按正式薪资）
2. **补偿金报税**：补偿金单独报税，不与工资合并申报，工资表中单独列示
3. **上月未报税金额**：包含上月因绩效迟发导致未报税、提前发放的奖金等
4. **贺金/慰问金**：合并进本月薪资报税【待确认具体标准】
5. **调休折算**：在离职月份折算扣款【待确认折算标准】
6. **考勤异常**：超标迟到/早退按员工手册规定折算为事假【待确认具体规则】
7. **在家打卡**：匹配家庭住址自动标记异常，需与部门领导核实

### 7.3 数据联动规则

- 人事信息登记后，自动同步到花名册、考勤表、薪资核算表
- 员工状态变更（转正/离职）自动反映到当月薪资核算
- 调薪记录自动归档到员工薪资档案

### 8.11 错误处理规范

> 所有错误提示必须使用**中文**，清晰说明哪里出错、可能原因、建议操作。

**错误分类与处理**：

| 错误类型   | 前端展示                 | 控制台日志        | 用户操作建议        |
| ------ | -------------------- | ------------ | ------------- |
| 网络错误   | "网络连接失败，请检查网络后重试"    | 记录完整错误堆栈     | 提供"重试"按钮      |
| 权限错误   | "您没有权限执行此操作，请联系管理员"  | 记录用户ID、操作、时间 | 提示联系 admin    |
| 数据验证错误 | "XXX 字段格式不正确，应为 XXX" | 记录验证失败详情     | 高亮错误字段，显示正确格式 |
| 业务规则错误 | "XXX 操作失败：XXX（原因）"   | 记录业务规则冲突详情   | 提示如何满足规则      |
| 系统内部错误 | "系统繁忙，请稍后重试"         | 记录完整错误堆栈     | 提供错误编号，便于排查   |

**错误处理原则**：

- 前端不得展示技术细节（如 SQL 错误、堆栈跟踪）
- 所有异步操作必须有 loading 状态和错误捕获
- 表单提交失败时，保留用户已填写的数据
- 批量导入失败时，生成错误报告 Excel，标注每行失败原因
- 控制台警告必须处理，不得遗留；造成问题的警告需修复，暂无法修复的记录到 README 已知问题中

**错误码规范**：

| 错误码范围     | 说明                  |
| --------- | ------------------- |
| 1000-1999 | 用户认证相关（登录、密码、Token） |
| 2000-2999 | 权限相关（无权限、数据范围限制）    |
| 3000-3999 | 数据验证相关（字段格式、必填项）    |
| 4000-4999 | 业务规则相关（核算流程、审核流程）   |
| 5000-5999 | 系统内部错误（数据库、文件、网络）   |
| 6000-6999 | 外部系统对接（钉钉、个税系统、易快报） |
